# 어드민 레이아웃 규격 (1920 기준)

어드민 화면은 **1920px 캔버스**를 기준으로 설계한다. 사이드바를 제외한 콘텐츠 영역을 먼저 확정하고,
그 안에서 헤더 · 섹션 컨테이너 · 12컬럼 그리드로 상세 페이지를 조합한다.
디자인 컨셉은 **Toss** — 저채도 회색 배경(`--ds-color-bgSubtle`) 위 흰 카드, 그림자 대신 1px 보더, 강조는 primary 하나.

| 항목 | 값 |
| --- | --- |
| 캔버스 | 1920 |
| 사이드바 | 240 (접힘 64) |
| 콘텐츠 폭 | 1920 − 240 = **1680** (접힘 시 **1856**) |
| 콘텐츠 좌우 패딩 | 40 → 실사용 **1600** (접힘 시 1776) |
| 그리드 | 12컬럼 × gutter 24 (컬럼 ≈ 111.3) |
| 상단 헤더 | 72 (브레드크럼 + 타이틀 2줄이면 104) |
| 섹션 카드 | radius `--ds-radius-lg`, 1px 보더, 섹션 간 간격 24 |
| 콘텐츠 최대 폭 | `full`=1600 · `lg`=1200 · `md`=768 (가운데 정렬) |

<br />

## 1. 캔버스 분해

체크박스로 사이드바를 접으면(240 → 64) 콘텐츠가 1680 → 1856으로, 실사용 폭이 1600 → 1776으로 늘어난다.

<CanvasDiagram />

<br />

## 2. 상단 헤더 (AdminTopbar)

타이틀만 있으면 **72**, 브레드크럼(또는 설명)이 붙어 2줄이면 **104**.
좌우 패딩은 콘텐츠 패딩(40)과 같아 본문 그리드와 좌측 정렬을 공유한다.

<TopbarDiagram />

<br />

## 3. 12컬럼 그리드 + 섹션 컨테이너

실사용 1600 안에서 12컬럼 × gutter 24 → 컬럼 ≈ 111.3px.
좁아지면 8컬럼(`< 1440`) → 4컬럼(`< 1024`)으로 폴백한다.

## 5. 상세 페이지 조합

### 좌 8 : 우 4 — 본문 + 요약

```tsx
import { PageContainer, PageSection } from '../ds/PageContainer/PageContainer'
import { AdminGrid, AdminGridItem } from '../ds/AdminGrid/AdminGrid'

<PageContainer maxWidth="full">
  <AdminGrid>
    <AdminGridItem span={8}>
      <PageSection title="기본 정보">{/* 폼 */}</PageSection>
    </AdminGridItem>
    <AdminGridItem span={4}>
      <PageSection title="요약">{/* 상태·메타 */}</PageSection>
    </AdminGridItem>
  </AdminGrid>
</PageContainer>
```

### 12 — 전체 폭 (표·대시보드)

```tsx
<PageContainer maxWidth="full">
  <AdminGrid>
    <AdminGridItem span={12}>
      <PageSection title="주문 목록" card={false}>{/* 표 */}</PageSection>
    </AdminGridItem>
  </AdminGrid>
</PageContainer>
```

### 6 : 6 — 2분할

```tsx
<AdminGrid>
  <AdminGridItem span={6} spanMd={4} spanSm={4}>{/* 좌 */}</AdminGridItem>
  <AdminGridItem span={6} spanMd={4} spanSm={4}>{/* 우 */}</AdminGridItem>
</AdminGrid>
```

### 상세 페이지 뼈대 — DetailLayout

본문 + 우측 사이드(360) + 하단 sticky 액션 바.

```tsx
import { DetailLayout } from '../ds/DetailLayout/DetailLayout'
import { Button } from '../ds/Button/Button'

<PageContainer maxWidth="full" gap="lg">
  <DetailLayout
    aside={<SummaryCards />}
    asideWidth="md"   // md=360 · sm=280
    sticky
    footer={
      <>
        <Button variant="secondary" size="md" label="취소" />
        <Button variant="primary" size="md" label="저장" />
      </>
    }
  >
    <PageSection title="기본 정보">{/* 폼 섹션 */}</PageSection>
    <PageSection title="상세 설명">{/* 폼 섹션 */}</PageSection>
  </DetailLayout>
</PageContainer>
```

## 6. 레이아웃 상수 (단일 소스)

수치는 **`src/ds/PageContainer/layout.css`** 한 곳에서만 관리한다.
색 · 타이포 · 여백 리듬은 디자인 토큰(`--ds-*`)을 쓰고, 아래 상수는 폭 · 높이 계산에만 쓴다.

```css
--admin-canvas-w: 1920px;
--admin-sidebar-w: 240px;
--admin-sidebar-collapsed-w: 64px;
--admin-content-w: 1680px;            /* 1920 − 240 */
--admin-content-collapsed-w: 1856px;  /* 1920 − 64  */
--admin-content-pad: 40px;
--admin-grid-columns: 12;             /* md 8 · sm 4 */
--admin-grid-gutter: 24px;
--admin-topbar-h: 72px;
--admin-topbar-h-stacked: 104px;
--admin-section-gap: 24px;
--admin-max-full: 1600px;
--admin-max-lg: 1200px;
--admin-max-md: 768px;
--admin-aside-md: 360px;
--admin-aside-sm: 280px;
```
