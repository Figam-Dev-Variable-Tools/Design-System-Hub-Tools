너는 이 저장소의 **표준 작업자**다. 시작하기 전에 `CLAUDE.md` 를 읽어라. 아래는 그 요약이 아니라
**이미 사고를 낸 항목들**이다. 여기 적힌 것을 어기면 그 작업은 실패로 간주한다.

## 🛑 절대 하지 마라

1. **git 상태를 바꾸지 마라.**
   `stash` · `checkout` · `restore` · `reset --hard` · `clean` · `rebase` · `merge` — 전부 금지.
   이 저장소는 **여러 에이전트가 하나의 워크트리를 공유**한다. 네 폴더만 되돌아가는 게 아니라
   **다른 에이전트 수십 개 파일의 미커밋 작업이 사라진다.** 실제로 53개 파일이 이렇게 날아갔다.
   되돌아간 파일도 **타입체크는 통과하기 때문에 아무도 알아채지 못했다.**
   - 기준선(변경 전 원본)이 필요하면 → `git show HEAD:<path>` (파일을 건드리지 않고 읽는다)
   - 백업이 필요하면 → `cp <path> <path>.bak`
   - 커밋·브랜치·푸시는 **오케스트레이터만** 한다. 하지 마라.
   - PreToolUse 훅이 실제로 막는다. 차단 메시지를 보면 우회하지 말고 **위 대안을 써라.**

2. **하드코딩 금지.** 색·크기·간격·폰트 리터럴(`#3D6BFF`, `16px`, `700`) 금지 → `var(--ds-*)`.
   사용자에게 보이는 **문구를 컴포넌트 안에 박지 마라** → `labels` prop(`src/shared/labels.ts`).
   매직넘버는 이름 있는 상수 + **왜 그 값인지** 주석.
   Figma 생성기도 같다 — **raw hex 금지**, Figma Variables 에 바인딩한다.

3. **이미 있는 걸 두고 새로 만들지 마라.**
   만들기 전에 `ls src/ds/` → `grep -rn "<비슷한이름>" src/ds/`.
   표·빈 상태·행 액션·툴바·확인창·배지·타임라인·업로드는 **이미 있다.**
   새 컴포넌트가 필요하면 **"왜 기존 것으로 안 되는지" 한 문장으로 설명할 수 있어야 한다.** 못 하면 만들지 마라.

4. **추측 금지 (할루시네이션).**
   prop·클래스·토큰·Figma 속성 이름을 기억이나 관례로 지어내지 마라. **항상 grep 으로 확인하고 인용하라.**
   없는 prop 을 쓰면 타입체크가 잡지만, 없는 **동작**을 가정하면 아무도 못 잡는다.
   `"아마 ~일 것"` 이라고 쓰고 싶어지는 순간이 바로 확인해야 하는 순간이다.

## ⚠️ 이미 일어난 사고 — 같은 함정을 밟지 마라

- **개명은 규약 준수인 동시에 사고다.**
  Figma 세트의 속성 이름을 바꾸면, 그 세트를 부르는 **화면의 `inst()` 오버라이드가 조용히 끊긴다**
  (`inst()` 는 없는 속성 이름을 **경고만 하고 무시**한다). `ProductCard.Price` 를 개명했더니
  상품 카드 10장이 전부 세트 기본가로 렌더됐는데 **5개 게이트가 전부 초록이었다.**
  → **속성을 개명했으면 그 세트를 호출하는 화면도 같은 커밋에서 고쳐라.** 그리고 `verify:screens` 를 돌려라.

- **재사용을 밀어붙이면 기능과 문구가 조용히 사라진다.**
  공용 컴포넌트로 갈아끼우다 "총 N건" 표시, 엑셀 버튼 라벨, 카드 뷰 일괄 액션 바가 사라진 적이 있다.
  → **재사용으로 무언가 사라진다면 그건 재사용이 아니라 축(prop) 부재다.**
  우회로(래퍼 div·sr-only·keydown 가로채기)를 만들고 싶어지면 **우회하지 말고 축을 추가하거나 보고하라.**

- **"고쳐서 통과"와 "안 봐서 통과"는 다르다.**
  게이트나 검사기를 새로 만들었으면 **결함을 일부러 주입해 실제로 잡는지 확인**하라.
  조용한 통과(파싱 실패를 warn 으로 넘기는 것 등)는 **실패로 취급**해야 한다.

- **주석이 "왜 이렇게 안 했는지"를 말하고 있으면 그 결정을 존중하라.**
  (`CustomerDetail` 에 "공용 Statistics 를 쓰지 않는다 — 좁은 aside 에서 금액이 잘린다"는 주석이 있었는데
  무시하고 바꿨다가 금액이 잘렸다.)

## ✅ 끝내기 전에 반드시

작업 폴더만 건드리고, **끝났다고 말하기 전에 게이트를 실제로 돌리고 출력을 봐라.**

```bash
npx tsc -p tsconfig.app.json --noEmit          # React 타입
cd figma-plugin && npx tsc --noEmit            # 플러그인 타입
node scripts/verify-parity.mjs                 # 스토리 ↔ 세트 존재
node scripts/verify-mapping.mjs                # prop → Figma 속성 타입
node scripts/verify-naming.mjs                 # 이름 파리티 (코드 == Figma)
node scripts/verify-screen-props.mjs           # 화면의 inst() 속성 실재 확인
```

## 보고 형식 (마지막 메시지가 곧 반환값이다 — 사람에게 말하듯 쓰지 마라)

- **변경**: 파일별 한 줄 요약
- **게이트**: 각 명령의 실제 결과 (통과/실패 + 실패 시 출력)
- **못 한 것**: 손대지 않은 것과 **그 이유** (추측으로 채우지 말고 모르면 모른다고 써라)
- **막힌 것**: 축이 없어서 우회로가 필요했던 지점 — 우회로를 만들지 말고 여기에 적어라
